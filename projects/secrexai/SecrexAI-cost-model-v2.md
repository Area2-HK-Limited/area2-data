## 🔬 Eric 實際用量分析（b03 session 數據）

**數據來源：** b03 對話記錄分析（2026-03 ~ 2026-04）

| 指標 | 數值 |
|------|------|
| **峰值用戶小時** | 2026-04-03 02:00（16 user messages / 小時）|
| **峰值總 events** | 74 events（包括 tool calls）|
| **平均每活躍小時** | 51.2 events |
| **活躍小時總數** | 68 小時 |
| **總 message 數** | 3,479 user messages |

### 每個 User Message 的 API Calls 估算

```
Peak hour: 16 user messages → 74 total events
每條 user message：約 74 ÷ 16 ≈ 4.6 events

估算 API calls：
- 1 user message → 1 API request（M2.7 對話）
- tools（搜尋/記憶讀取）→ ~1-2 API requests

結論：每 user message ≈ 2-3 API requests（保守估算：2 calls）
```

## 📊 用量假設及 API 成本計算（USD）

### 用量假設（3 個 Scenario 比較）

| 指標 | Scenario 1：峰值全日 | Scenario 2：適中活躍 | Scenario 3：正常活躍 |
|------|---------------------|---------------------|---------------------|
| **msg/hr** | 16（峰值）| 8 | 4 |
| **msg/日（9小時）** | 144 | 72 | 36 |
| **msg/月** | 4,320 | 2,160 | 1,080 |
| **API calls/月** | 8,640 | 4,320 | 2,160 |
| **M2.7 API cost/人/月** | $7.78 | $3.89 | $1.94 |

> **基礎：** Eric 峰值 16 msg/hr × 9 小時；三個 scenario 分別以 16/8/4 msg/hr 計算全日 9 小時

### 每月 API 消耗計算（USD）

#### MiniMax M2.7-highspeed（主要對話）

```
每 call 消耗：1,000 input + 500 output tokens
每 call 成本：(1,000 × $0.30 + 500 × $1.20) / 1M = $0.0009/call

 Scenario 1（峰值）：8,640 calls × $0.0009 = $7.78/人/月 → 50人 $389/月
 Scenario 2（適中）：4,320 calls × $0.0009 = $3.89/人/月 → 50人 $194/月
 Scenario 3（正常）：2,160 calls × $0.0009 = $1.94/人/月 → 50人 $97/月
```

#### Qwen3-Omni Flash Realtime（STT）

假設 20% 用戶使用語音功能，平均每人 10 分鐘語音/月：
```
每分鐘成本：$2.709/M × 900 tokens = $0.00244/分鐘

50 用戶每月：50 × 20% × 10分鐘 × $0.00244 = $0.24/月
```

#### Qwen-Plus（複雜分析）

假設每人每月 10 次複雜分析，每次 10K input + 5K output：
```
每 call 成本：(10K×$0.40 + 5K×$1.20)/1M = $0.01/call

50 用戶每月：50 × 10 × $0.01 = $5.00/月
```

#### Perplexity Sonar（搜尋）

假設每人每月 30 次搜尋：
```
每 call：$5/1,000 = $0.005/次

50 用戶每月：50 × 30 × $0.005 = $7.50/月
```

#### Nano Banana Pro（圖片生成）

假設 Starter 平均 5 張/月，Business 平均 15 張/月：
```
Starter（40 人）：40 × 5 × $0.134 = $26.80/月
Business（10 人）：10 × 15 × $0.134 = $20.10/月
圖片合計：$46.90/月
```

## ⚠️ Rate Limit 風險評估

### 50 用戶峰值分析

```
50 用戶同時峰值：
- 每用戶 16 messages/hr × 50 = 800 messages/hr
- 每 message 2 API calls = 1,600 calls/hr

Max-Highspeed capacity：15,000 calls / 5小時

5小時內峰值窗口總消耗：1,600 × 5 = 8,000 calls
8,000 < 15,000 ✅ 峰值安全！

理論可支撐峰值用戶數：
15,000 ÷ (16 × 2) = 468 峰值用戶
```

### 不同用戶場景評估

| 場景 | 用戶數 | 可行性 | 建議計劃 |
|------|--------|--------|----------|
| **10 用戶** | 10 | ✅ 非常安全 | Max-Highspeed |
| **50 用戶** | 50 | ✅ 安全 | Max-Highspeed |
| **100 用戶** | 100 | ✅ 安全 | Max-Highspeed |
| **200 用戶** | 200 | ⚠️ 緊張 | 考慮升級至 Ultra |
| **500 用戶** | 500 | ❌ 需升級 | Ultra-Highspeed 或多帳號 |
