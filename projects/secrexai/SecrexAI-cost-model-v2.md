## 🔬 Eric 實際用量分析（b03 session 數據）

**數據來源：** b03 對話記錄分析（2026-03 ~ 2026-04）

| 指標 | 數值 |
|------|------|
| **峰值用戶小時** | 2026-04-03 02:00（16 user messages / 小時）|
| **峰值總 events** | 74 events（包括 tool calls）|
| **平均每活躍小時** | 51.2 user messages |
| **活躍小時總數** | 68 小時 |
| **總 message 數** | 3,479 user messages |

### 每個 User Message 的 API Calls 估算

```
Peak hour: 16 user messages → 74 total events
每條 user message 約產生：74 ÷ 16 ≈ 4.6 events

估算 API calls：
- 1 user message → 1 API request
- 1 assistant reply → 1 API request  
- tools（搜尋/記憶讀取）→ ~1-2 API requests

結論：每 user message ≈ 2-3 API requests
保守估算：3 API requests / message
```

### 峰值用戶 API 消耗

```
Peak user: 16 messages/hr × 3 API calls = 48 requests/hr
Max-Highspeed capacity: 15,000 requests / 5小時 = 3,000 requests/hr

峰值用戶 capacity 百分比：48 / 3,000 = 1.6%
```

## 💰 50 用戶成本計算

### 假設用戶分佈

| 計劃 | 用戶數 | 平均 messages/月 | API calls/月/人 |
|------|--------|-----------------|-----------------|
| **Starter** | 35 人 | 30 messages | 30 × 3 = 90 calls |
| **Business** | 15 人 | 100 messages | 100 × 3 = 300 calls |

### 每月 API 消耗計算

**Starter（35 人）：**
```
35 × 90 calls = 3,150 calls/月
```

**Business（15 人）：**
```
15 × 300 calls = 4,500 calls/月
```

**50 用戶總消耗：**
```
3,150 + 4,500 = 7,650 calls/月
```

### Max-Highspeed 配額覆蓋分析

```
Max-Highspeed: 15,000 requests / 5小時滾動窗口

每 5 小時：15,000 calls
每日（24小時）：15,000 × 4.8 = 72,000 calls（理論）
每月：72,000 × 30 = 2,160,000 calls（理論）

50 用戶每月消耗：7,650 calls
覆蓋率：7,650 / 2,160,000 = 0.35% ✅

即使峰值：50 users × 48 calls/hr = 2,400 calls/hr
5小時峰值窗口：2,400 × 5 = 12,000 calls
12,000 < 15,000 ✅ 仍然足夠！
```

> **結論：Max-Highspeed ($80/月) 完全足夠支撐 50 用戶 ✅**
> 即使所有用戶同時達到峰值，15,000 requests / 5小時 limit 仍然覆蓋得住（12,000 calls < 15,000）

## ⚠️ Rate Limit 風險評估

### 50 用戶峰值分析

```
50 用戶同時峰值：
- 每用戶 16 messages/hr × 50 = 800 messages/hr
- 每 message 3 API calls = 2,400 calls/hr

Max-Highspeed capacity：15,000 calls / 5小時

5小時內峰值總消耗：2,400 × 5 = 12,000 calls
12,000 < 15,000 ✅ 峰值安全！

理論可支撐峰值用戶數：
15,000 ÷ 48 calls/hr per user = 312 峰值用戶
```

### 不同用戶場景評估

| 場景 | 用戶數 | 可行性 | 建議計劃 |
|------|--------|--------|----------|
| **10 用戶** | 10 | ✅ 非常安全 | Max-Highspeed |
| **50 用戶** | 50 | ✅ 安全 | Max-Highspeed |
| **75 用戶** | 75 | ⚠️ 緊張 | Max-Highspeed 或 Ultra |
| **100 用戶** | 100 | ❌ 需升級 | **Ultra-Highspeed** |
| **200 用戶** | 200 | ❌ 需升級 | **Ultra-Highspeed** |

### Ultra-Highspeed 升級門檻

當用戶超過約 **75 人**時，考慮升級至 Ultra-Highspeed ($150/月)：
- 30,000 requests / 5小時（2倍容量）
- 可支撐最多 625 峰值用戶
