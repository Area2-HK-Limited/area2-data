## 一、行政 / 會計日常工作地圖

### 1.1 洗樓王這類街頭營銷公司的行政 / 會計職責

洗樓王屬於項目型服務公司，行政 / 會計的工作範圍比一般後勤部門更闊，幾乎滲透到每個項目環節：

| 職責範疇 | 具體工作 |
|----------|----------|
| **項目立項與成本管理** | 開新項目、登記客戶、分配預算、追蹤實際成本 |
| **發票管理** | 建立 Invoice、發送給客戶、跟進付款狀態、沖銷 |
| **請款與收款** | 監控逾期應收帳款、聯絡客戶催款、整理入數紀錄 |
| **供應商付款** | 核對外判 crew 費用、、印刷成本、雜項支出 |
| **項目收支報告** | 核算每個項目的毛利率、計算員工提成、整理月結數據 |
| **內部協調** | 通知客戶付款進度、回覆客戶查詢、安排 crew 檔期 |
| **行政庶務** | 合同管理、文件歸檔、基本文書 |

### 1.2 有 Invoice Ninja Project Module 後，行政 / 會計的痛點

Invoice Ninja 解決了「記錄」問題，但以下 moment 依然痛：

| 痛點場景 | 問題描述 |
|----------|----------|
| **發票建立了，但客戶遲遲未付** | 無人主動追，應收帳款愈積愈多 |
| **項目執行中，不知道用了多少預算** | 要登入系統、被迫自行查找數字，費時 |
| **跨項目成本對比** | 多個項目同時進行時，很難快速口述現況 |
| **月結前加班盤點** | 大量數據要人手整理，月底壓力集中 |
| **客戶來電問「筆錢幾時到期？」** | 要立即查系統，但同事可能在會議中 |
| **項目超支發現得太遲** | 成本錄入是事後的，沒有人即時預警 |

**核心矛盾：** Invoice Ninja 是「被動資料庫」——你有空去睇，先知發生咩事。但行政 / 會計的工作本質係「被需求打擾」，被動工具永遠追不上主動查詢。

## 三、唔同於其他工具的價值

### 3.1 點解係 WhatsApp 而唔係直接登入 Invoice Ninja？

| 維度 | 直接操作 Invoice Ninja | SecrexAI（WhatsApp） |
|------|------------------------|----------------------|
| **啟動速度** | 要搵電腦 / 搵 login、揀項目、等頁面 load | 拿起手機、問完即走 |
| **隨時可用** | 坐在電腦前先做到 | 搭車、行緊路、見緊客都可以 |
| **多人同步** | 要分享帳號或給予權限 | 每人有自己 WhatsApp，直接問 |
| **老闆查數** | 要 admin 幫手查 | 老闆自己 WhatsApp 問就得 |
| **突發查詢** | 要記得喺邊個 menu | 直接用中文描述，唔使記功能 |
| **跟進連貫性** | 查到數字仲要自己做後續 | AI 可以幫你主動提醒或催款 |

### 3.2 邊個場景下 WhatsApp 指令比 direct 操作更方便？

✅ **絕對優勢場景：**
- 老闆行過問你「個項目點啊？」—— 即刻口述，唔使郁手
- 客戶打電話來問「筆錢幾時到期？」—— WhatsApp 秒回
- 會計趕住出月結報告——語音 input，一個指令整理好
- 多個員工同時需要查不同項目數據——唔使搶 account login

⚖️ **兩者配合才是最佳狀態：**
- SecrexAI 處理「問與答」的即時需求
- Invoice Ninja 處理「記錄與管理」的底層數據
