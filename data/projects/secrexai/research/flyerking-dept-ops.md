## 1. 推廣團隊日常工作地圖

### 1.1 前線推廣員 / 項目執行同事做咩？

| 階段 | 典型任務 | 涉及動作 |
|------|----------|----------|
| **事前準備** | 物資點收、分袋、包裝 | 執貨、計數、標籤 |
| **出發前確認** | 領取物資、確認地點時間 | 聯絡督導、影相確認 |
| **實地執行** | 派傳單、擺設攤位、活動示範 | 派發、記錄數量、影片拍攝 |
| **客戶聯絡** | 匯報进度、反映問題 | WhatsApp / 電話通知 |
| **事後匯報** | 填寫工作日誌、統計數字 | 表格、回傳相片/影片 |
| **物資交還** | 歸還剩餘物資、核對庫存 | 盤點、填報損耗 |

### 1.2 現有痛點

| 痛點 | 具體情況 | 影響 |
|------|----------|------|
| 物資分配不清晰 | 前線同事唔知邊度有現貨、邊件係邊個預咗 | 出發先發現漏帶，重複來回 |
| 客户臨時改要求 | 臨時加單、加地點、調整數量 | 前線同事難以及時更新，督導來回確認費時 |
| 活動後統計麻煩 | 派發數量靠人手估計、WhatsApp 群洗板 | 數據散落，事後難以整合出準確報告 |
| 督導無法即時掌握前線情况 | 前線同事 WhatsApp 通知口語化，格式不統一 | 管理層要另外執靚先可記入 Invoice Ninja |
| 物資損耗責任不清 | 損耗無即時記錄，事後爭拗 | 影響客戶 billing 準確性 |

### 1.3 Invoice Ninja Project Module 之後，仲有咩未解決？

Invoice Ninja 解決了 **項目收支追蹤** 和 **客戶帳單管理**，但前線同事無法直接使用——原因：

- Invoice Ninja 係 web-based admin 工具，不適合前線即時操作
- 前線同事主力靠 WhatsApp 溝通，唔會特意登入系統
- 仍然需要 admin 同事幫手將 WhatsApp 消息轉化成系統記錄

**核心缺口：前線與系統之間的「最後一公里」——WhatsApp 即時輸入 + AI 自動整理入系統。**

### 4.1 唔使記嘢 → 問 AI 就知

| 之前 | 之後 |
|------|------|
| 要記住邊件物資喺邊、有幾多 | 直接 WhatsApp 問 SecrexAI，5 秒有答案 |
| 要打電話問督導先知明日地點 | WhatsApp 一句搞掂 |
| 要等人事同事回覆才知道庫存 | AI 即時讀取庫存數據 |
| 要排隊等 admin 同事幫手查資料 | 自己 WhatsApp 直接問 |

### 4.2 唔使 admin 同事幫手查 → 自己 WhatsApp 搞掂

**實際節省：**
- 前線同事每次查詢平均省 15-30 分鐘來回協調時間
- Admin 同事少處理大量「幫我查下…」的 WhatsApp 訊息
- 督導減少被前線重複打擾，可專注更高價值的協調工作

### 4.3 前線同事使用門檻

| 條件 | 說明 |

| 只需要 WhatsApp | 唔使下載新 App，唔使登入任何系統 |
