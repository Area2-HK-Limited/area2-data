## 五、現金流運作模式

```
Area2 每月出糧給員工（含社保）
        ↓
員工收到糧後回水返 Area2
        ↓
每半年政府發放一次資助
        ↓
Area2 收到資助款
```

### ⚠️ 注意
- 需要**先墊資**出糧 + 社保，半年後先收到資助
- 最差情況：前 6 個月墊資 = (¥4,500 + ¥1,500) × 6 = **¥36,000**

## 六、Maverick 分成方案

### 背景
- Maverick 係 Hostlink 正職同事，以 Area2 員工身份借資料申請
- 佢會知道每人資助金額（因為出糧→回水流程透明）
- 佢唔需要做法人代表，唔承擔法律責任
- 需要配合 3 年（持續出糧→回水循環）
- 由於前海資助屬**半年發放一次**，而且實際到帳時間可能有延遲，因此不適合採用高額一筆過付款模式

### 最新建議方案（推薦）

**方案 A：小額啟動費 + 實收分成**

| 項目 | 建議 |
|------|------|
| 每人頭預計 3 年總資助 | **¥110,000** |
| Maverick 分成比例 | **20%** |
| Maverick 理論總收入 | **¥22,000 / 人頭** |
| 前期啟動費 | **¥2,000 / 人頭** |
| 後續分成 | **餘下 ¥20,000 按每次實際收到資助款再分配** |

### 付款邏輯
- 先支付 **¥2,000 / 人頭**，作為合作正式開始的誠意金 / 啟動費
- 其後不承諾固定金額或固定每期派發數字
- **必須按 Area2 每次實際收到的前海資助款項，再按協議比例分成**
- 若政府延遲發放、分批發放、或最終金額少於預期，則按**實收金額**重新計算

### 原因
- 對 Maverick：前期有回報，知道合作已開始，不是空口承諾
- 對 Area2：避免在未收到資助前就承擔過高現金流壓力
- 對雙方：將付款節奏綁定實際收款，減少未來爭議

### 條款建議
- 分成基礎：**只按實際收到資助款項計算**
- 不按：申請金額 / 預計金額 / 批核金額作結算依據
- 結算時間：每次資助到帳後 **14-30 日內** 進行分配
- 配合期：Maverick 需按協議配合整個申請及後續收款周期

### 備註
- 以 3 年、半年一結計算，後續款項理論上約分 6 期，但**不建議寫死每期固定金額**
- 實際每期派發金額應以當次資助實收款按比例計算
